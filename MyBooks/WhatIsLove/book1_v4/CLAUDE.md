## ALWAYS: show a nicely formatted diff before applying any edit

Before writing ANY change to a chapter or file, ALWAYS preview it as a **nicely formatted, styled diff** — a rendered, track-changes redline (removed text struck through on light red, added text on light green), not plain markdown strikethrough in chat and never a raw/horizontally-scrolling code diff. In this environment, render it with the visualization widget so it reads like the page. Then wait for Patrick's approval ("y"/"n") before saving. This is non-negotiable and applies every single time, even for one-line changes. (Full redline spec is in agents.md / §1.)
